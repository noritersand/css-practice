grid-template-rows

개요

grid-template-rows는 행의 개수와 높이를 정의할 때 사용한다.

Grid 컨테이너에 직접 지정하며, 기본값은 none이다.

Syntax

Values

grid-template-rows는 특수한 경우가 아니면 보통 grid-template-columns와 같이 사용한다.

grid-template-rows만 단독으로 지정하면 명시적 행의 개수와 높이만 정해지고, 열의 크기는 지정하지 않은 상태가 된다. 자동 배치 과정에서 필요한 암시적 열의 너비는 기본적으로 auto로 처리된다.

px

행의 높이를 고정 수치로 지정하는 방식.

fr

행 트랙이 차지할 높이를 비율로 지정하는 방식

fr은 Grid Layout에서만 쓰이는 특수한 단위로, Grid 컨테이너 내부에서 고정 크기를 가진 요소들을 제외한 남은 공간(leftover space) 중 Grid Track이 차지할 비율을 나타내는 값이다.

repeat()

동일한 높이의 행을 반복 지정할 때 사용한다. 첫 번째 인수는 반복 횟수, 두 번째 인수는 반복할 행 높이를 의미한다.

auto

auto는 콘텐츠 크기를 기준으로 행의 높이를 자동으로 결정한다. 컨테이너에 여유 공간이 있으면 정렬/트랙 크기 계산에 따라 늘어날 수 있다.

1
2
3

minmax()

행의 최소 높이와 최대 높이를 범위로 지정한다. 첫 번째 인수가 최소 높이, 두 번째 인수가 최대 높이다.

이 값을 지정하면 컨테이너의 높이 변화에 따라 행 트랙의 높이가 지정한 범위 안에서 커지거나 줄어든다.

1
2
3

auto-fill, auto-fit

컨테이너의 높이에 따라 행의 개수를 자동으로 줄이거나 늘린다.

auto-fill과 auto-fit은 컨테이너에 공간이 남을 때 차이가 발생한다. auto-fill은 빈 행 공간을 그대로 유지하지만, auto-fit은 빈 행을 축소시켜 기존 행들이 남은 공간을 넓게 나누어 갖도록 처리한다.

가령 높이 1000px의 컨테이너에 아이템이 2개고 다음처럼 지정했을 때:

auto-fill은

  • gap이 없다고 가정할 때, 컨테이너에 200px짜리 행이 최대 5개 들어갈 수 있음을 계산한다.
  • 아이템 2개를 첫 번째, 두 번째 행에 넣고 남은 3개 공간을 빈 행으로 계속 유지한다.
  • 5개의 행이 컨테이너 높이를 나누어 가지므로 각 행은 약 200px 높이가 되고, 아이템은 위쪽 행에만 배치된다.

auto-fit

  • 똑같이 gap이 없다고 가정하고, 5개의 행 공간을 계산하지만, 아이템이 들어가지 않은 빈 행 3개의 높이를 0px로 축소시킨다.
  • 남은 3개 행이 사라졌으므로, 실제 존재하는 2개의 아이템이 남은 1000px 공간을 1fr : 1fr 비율로 500px씩 나누어 가진다.

⚠️ 이 값들은 repeat() 내부에서만 유효하다.
